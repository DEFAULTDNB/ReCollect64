Minecraft Engine in SM64!

Built Different

ROM hacker and modder Arthurtilly has today released a demonstration of his voxel engine running within Super Mario 64. Unbelievably, the engine runs within the 8 MB RAM of the Nintendo 64, albeit at a slightly lower draw distance, and it also works emulators.

As an active member of the SM64 ROM hacking community, he has hosted three hacking contests as well as creating twelve ROM hacks, including SM64 The Green Comet: Green Comet and Gravity Swap 64, and has co-authored at least seven projects, such as Mario Builder 64 with Rovertronic and Serene Fusion with authors including BroDute, scuttlebug_raiser, usernamesarespiders, Luigiman04, and TheGael95.

 

This is a showcase of a Minecraft voxel engine I developed completely within the SM64 engine and can run on a real Nintendo 64 console. It uses multithreading to be able to seamlessly generate terrain and meshes over multiple frames without causing any lag.

It currently supports an infinite world, multiple block types and shapes (easily extensible), multiple biomes with unique noise parameters and smooth blending together, the beginning of support for fluids like water and lava, and can support smooth lighting and ambient occlusion.

In the future I hope to support more features such as:

+ Flowing water and lava

+ A proper lighting system and day/night cycle

+ Saving and loading of worlds (even on console)

+ Terrain features such as trees and vegetation to make the world more lively

+ An inventory system

Crafty Creations

The video showcases infinitely procedurally generated terrain, placement and destruction of blocks, ambient occlusion with lighting, half-height blocks (slabs), basic material cycling (grass, stone, brick, water, sand, lava, ice, etc), water blocks that you can swim through, smooth transitioning textures, alternating music per biome (like Banjo-Kazooie), the ability to make massive-scale mountains, valleys and canyons, and to top it off, it has a debugger built in!

With a separate CPU thread handling terrain generation from the chunk (block) loading, the only bottleneck for creating lag on real hardware stands with the rendering. Arthurtilly's engine prioritises generating optimised meshes, rather than calculating the meshes, to reduce stutter and prevent too much lag from loading/generating blocks.

Using the full featured debugger that's also been built in, you can also see detailed statistics from the RDP (the Reality Display Processor for 2D and polygon rendering), RSP (the Reality Signal Processor for graphic/audio data transform), the current CPU load stress and a nifty FPS readout.

There has also been a custom memory allocator designed especially for this project, which oversees chunks, lightmaps, GFX and VTX management, allowing it all to run impressively smoothly while generating literally thousands of chunks at any one time!

With features planned such as trees and vegetation, enemies and objects, an item inventory system, SD card support for more expansive worlds on flashcarts, and the ability to save and load your worlds, this project is extremely ambitious and already very impressive.

There is no planned release date yet, but you'd better be sure that I'll be all over this whenever it eventually drops!
