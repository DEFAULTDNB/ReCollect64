Decomp Vs Recomp Vs Ports

An Introduction

Because it's a confusing subject for those who aren't in the know, I wanted to produce a post that clarifies and rectifies some information regarding the differences between a decompilation, a recompilation and a port.

I recently fell into the trap of joining the dots between various projects and came a cropper by speculating far too much on things presented online. This post aims to pin down the crucial nuances for both myself and anyone else reading my content.

Decompilation

Firstly, decompilation is the process of creating a source code base in a high-level programming language directly from the assembly code found within a game's ROM. The primary goal of this is so that the game can be preserved, modified and studied in greater detail.

Speedrunners and modders can use this codebase to extract copious amounts of intricate information, using it to produce new strategies or even control features and functions to enhance the game or change it entirely.

Super Mario 64 and The Legend of Zelda: Ocarina of Time are prime examples of games that have countless mods and fixes applied to them now that they have been decompiled to a point where information is easily editable.

The process of manually decompiling can take years, but as mentioned, a full decompilation is not strictly required to produce a working effigy of the original. On the contrary, GoldenEye 007 is actually only decompiled to around 60%, yet SonicDcer and KholdFuzion were able to construct a working game from the code base using N64Recomp.

Fundamentally: The decompilation of a game does not guarantee that a PC port will be produced.

Most communities who are decompiling ROMs are not doing it with that goal in mind. Porting is a fantastic byproduct of the process, but it is something that we should never take for granted.

Recompilation

Nintendo 64: Recompiled, known as "Static Recompilation", is a radical project, led by Mr-Wiseguy, that can essentially generate C programming language from a disassembly of a game ROM, ready to be compiled for any platform. The recompiled code runs on a reimplementation of the SDKs used by the game called N64ModernRuntime, which is comprised of two libraries, ultramodern and librecomp.

From there a port creator can choose a graphics renderer to translate 3D data into a 2D display on the end user's screen. To date, all public recompilation projects have incorporated Dario's RT64 renderer, which offers a range of enhancements, such as widescreen and high framerate support, and it will eventually include ray tracing too! Wiseguy recently demonstrated that the general framework for utilising a recompilation's output has been improved to run on lower-spec hardware and with modern controller support, and it is linked to a host that facilitates the downloading of cheats and mods within the final executable.

Recompilation projects can take advantage of game decompilations to produce mods or improve the chosen game, but the two are quintessentially unrelated.

The performance accuracy is reportedly comparable to the paraLLEl-RDP project, which itself aims to emulate the Nintendo 64's RDP graphics chip as accurately as possible using Vulkan compute.

Each recompilation requires manual fixes per game to iron out any wrinkles in the translation process, and as a result, each game requires its own executable. All end users will have to do is provide the ROM to start the game.

The best examples of public recomps available today include Dinosaur Planet, Goemon and Majora's Mask, though other games such as Super Smash Bros., Kirby 64: The Crystal Shards, Chameleon Twist, Banjo-Kazooie, Rocket: Robot on Wheels, Superman 64 and Quest 64 are in the works!.

While N64Recomp games are technically ports, the term "recompiled" was chosen to make the distinction between the exact processes used in crafting them.

Porting

Using a decompilation's codebase, it is possible for games to be ported to other platforms such as PC, Vita, Android, 3DS, Nintendo Switch and even the PlayStation 2 and Dreamcast!

As witnessed through the efforts of The Harbour Masters, Ship of Harkinian, 2 Ship 2 Harkinian, and most recently Starship have given The Legend of Zelda: Ocarina of Time, The Legend of Zelda: Majora's Mask and Star Fox 64 fresh leases of life by way of PC ports. These ports offer faithful renditions of the games but also allow for features such as randomisation, asset swapping and advanced statistical tracking.

The core library that facilitates the ports listed above is known as Libultraship by Kenix3 and over 80 contributors. Other ports, such as Super Mario 64 and Perfect Dark, make use of alternative SDK libraries, but essentially, all ports use a compiler such as GCC, MSVC or Clang, for example (programmer's choice), to build the executable from the raw decompiled code base, incomplete or otherwise, as shown in the handy diagram above.
