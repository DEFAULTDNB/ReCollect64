N64Memtest 0.9 Released

By Rasky ~ Jun 11, 2026

A Hunk of Burn-in Love

With the Nintendo 64 making its 30th journey around the sun this year—yes the console is turning 30 in Japan on the 23rd of June—it's always good to be able to give the system a thorough testing to ensure it's operating correctly. N64Memtest by Rasky provides an intuitive tool that runs directly on your console via flashcart that aims to stress test the on-board 4MiB Rambus DRAM.

We have been able to run diagnostics on general hardware, displays, and controllers for a while, thanks to tools like Artemio's 240p Test Suite, Wermipls Mimi, and Ryzee119's Controller Test ROM, as well as Meauxdal's N64 Revision Tester, but a full-on MIPS-based memory tester is a much-welcomed addition.

 

N64Memtest is a homebrew memory diagnostic tool for the Nintendo 64. It exercises RDRAM with a suite of classic RAM test patterns and March algorithms, reports mismatches with bank/address details, and keeps the UI responsive while tests run in hand-written MIPS assembly.

Features:

18 memory test patterns, including:

+ Random value, checkerboard, solid bits

+ Walking ones / walking zeroes

+ Sequential increment, bit spread, bit flip

+ March MATS+, MATS++, A, B, C-, C+, SR, SS, X, Y

Four test profiles

+ Quick — fast sanity check (random, checkerboard, solid bits, selected March tests)

+ Full — comprehensive coverage (all March tests, walking patterns, bit flip, random)

+ Burn-in — continuous stress run (Full plus every remaining pattern; infinite loops)

+ Custom — pick banks and tests individually

Configurable loop count — 1, 5, 20, or infinite (Burn-in always runs continuously)

Real-time UI — progress percentage, KiB tested, slice count, error log with address/expected/actual values

Pause / resume / stop controls during a run

Snake easter egg — play while waiting for long burn-in sessions

Headless burn-in mode — if no controller is connected on port 1, the tool auto-starts a burn-in run (useful for unattended testing)

iQue Player support — detects iQue hardware and tests the 16 MiB physical RAM window correctly

Debug logging — structured log messages over emulator log and USB debug

Just pop it onto a flashcart, and run the tests. If you find that you have black screens, graphical glitches, or crashing on your console in general, it might be time to look into fixing or replacing the RDRAM, or even procuring another console, if the issues make gameplay unbearable.

A tool like this can also be taken with you to any place you might want to go to purchase an N64 console. If you buy from local sellers, pawn shops in town, or even retro gaming market events, bring this along with you to ensure you're making a sound purchase of a console with no issues.
