Doom-N64 Released

Blow Your WAD

A couple of days ago I spotted a project that slightly blew my mind. Typically, when playing Doom on the Nintendo 64, you're limited to playing the 1997 retail release or prebuilt modified ROM's that may contain outdated mods that you wish included other levels or WADs.

Nintendo 64 homebrew developer Ham1ton, aka Hamltoe, has released a stunning new project that gives you full instructions on how to build your own custom Doom 64 with any WADs you wish, but he also has another neat surprise: this version allows for full 2, 3, and 4-player multiplayer as well as cooperative play!

 

DOOM-N64

DOOM-N64 is a Windows-friendly porting project for running classic DOOM on the Nintendo 64. Play your favorite mods using the WAD browser to load IWAD and PWAD files at runtime. Try the original shareware demo on the releases page.

What This Project Does:

+ Builds a Nintendo 64 DOOM ROM on Windows using WSL.

+ Lets you choose IWAD and PWAD files in an in-game WAD browser.

+ Copies all .wad/.WAD files from WADs/ into ROM filesystem during build.

READ ME

In order to build your own WAD-packed version and enjoy the thrills of split-screen multiplayer on your own console, you will need to install the Libdragon N64 SDK toolchain installed at /opt/libdragon, Windows 10 or 11, and WSL2 with Ubuntu (or a similar distro).

Once you have the development environment set up and ready, you can download the project and alter it to your liking with your legally owned WAD files by following these instructions:

 

1) Get project

If cloning new:

git clone --recurse-submodules

cd DOOM-N64

If already cloned:

git submodule update --init --recursive

2) Add your WAD files

Place your legally owned files into the WADs/ folder.

Minimum setup: At least one IWAD (example: DOOM.WAD, DOOM1.WAD)

Custom mod setup: One IWAD + one or more PWAD files

Example:

  WADs/DOOM.WAD

  WADs/plutonia.wad

3) Build ROM

Fastest in VS Code:

  1, Run Task

  2, Select build-doom

Debug build with logging:

  1, Run Task

  2, Select build-doom-debug

Post-build verification:

  1, Run Task

  2, Select verify-wsl-n64-env

Important:

Run build and verify tasks sequentially (not concurrently).

All project tasks are pinned to N64_INST=/opt/libdragon and N64_GCCPREFIX=/opt/libdragon.

Command-line build in WSL (same result):

  export N64_INST=/opt/libdragon

  export N64_GCCPREFIX=/opt/libdragon

  make -j"$(nproc)"

  echo rom_bytes="$(stat -c%s Doom-N64.z64)"

4) Find output

ROM output: Doom-N64.z64

5) Launch and play

At startup, WAD browser appears:

IWAD to play base game.

PWAD to play mod. Browser asks for base IWAD.

For those without technical knowledge but who still wish to give it a go, the GitHub site offers a shareware option too, so there is no reason not to at least fire this up and give it a bash on your flashcarts.

I recall asking Immorpher a while back about the possibility of loading WADs directly into a ROM from the flashcart's SD card, and the answer was extremely positive, but as far as I know, no one has attempted to implement this yet. If it did, there would be no need for recompiling the ROM and all the more reason to get playing with friends!
