Project Brief: Duke Nukem: Zero Hour Fullscreen Netplay Recompilation by Snake Plissken

1. Overview: What It Is

This project is a modern, high-performance PC port of the 1999 Nintendo 64 classic Duke Nukem: Zero Hour, built utilizing the groundbreaking N64Recomp framework. Rather than running via traditional emulation, the original game’s machine code has been statically recompiled into native x86 C++ code.

The crowning achievement of this specific project is the complete extraction and overhaul of the game's native local split-screen code, transforming it into a modern, authoritative host/client network multiplayer experience where each player enjoys a full, uncompromised widescreen display.

2. The Technical Sorcery: How It Was Accomplished

Low-Level Netcode Integration: To bridge the network layer seamlessly, the high-performance ENet (reliable UDP) library was integrated directly into the recompiled N64 input controller layer (osContGetReadData).

Authoritative Input Architecture: The multiplayer operates on a strict Host/Client relationship. Instead of synchronizing massive, volatile game states across the network, the Client streams its raw controller inputs to the Host. The Host aggregates all inputs into a master state array, processes the "absolute truth" of the game loop, and broadcasts that state back to all connected clients at a locked 60Hz.

MIPS Graphics Pipeline Manipulation: Natively, triggering multiplayer forces the N64 engine to squish the display into a local split-screen. To break this limitation, surgical overrides were written into the recompiled graphics pipeline (patches.c). The runtime dynamically queries the network layer for the local player's index and intercepts the native N64 viewport and hardware scissor registers (gDPSetScissor).

The Fullscreen Viewport Hack: The engine is commanded to upscale the local player's specific quadrant to fill the entire monitor. Simultaneously, a custom fixed-point math routine intercepts the remote phantom player's scissor box, safely crushing their rendering field down into a legal 2x2 micro-viewport out of sight. This tricks the retro engine into rendering multiplayer, while providing both host and client a modern, isolated fullscreen experience.

3. Current State: What Is Fully Functional Right Now

Native PC Performance: Flawless 60Hz gameplay running natively on modern PC hardware, free from the frame-rate chugs and hardware limitations of the original N64 console.

Fully Realized Fullscreen Widescreen Netplay: Host and client can connect seamlessly over LAN or internet via ENet, ready up in the menus, and drop into multiplayer arenas with zero visual splitting or screen stretching.

Locked Synchronization: Complete inputs are mirrored effortlessly across devices, ensuring real-time response times vital for a fast-paced late-90s shooter.

4. The Horizon: What Is To Come

Campaign Co-Op Mode: Natively, Zero Hour never possessed a cooperative campaign mode. Work is currently underway to intercept the single-player level-loading sequence, forcing the main game loop to spawn multiple player indices and map objects without defaulting to Deathmatch rules.

Enemy AI Tracking Patches: Because original campaign enemies are hardcoded to track only a single player target (gPlayer[0]), custom MIPS routine hooks are being designed to recalculate enemy distance arrays, allowing the horde to dynamically track and hunt whichever Duke is closest.

Real-Time PBR/RTX Material Forge: Future development plans include exploring runtime hooks linked to local AI material generators (like ComfyUI). This layout aims to analyze the classic low-poly textures on the fly and generate modern PBR material data (normals, roughness, emissive paths) to feed directly into advanced Ray-Tracing render backends (RT64), bridging 1999 geometry with modern lighting. only in development however, for such effects will be backed into the final build and togglable via a next gen on and off button
